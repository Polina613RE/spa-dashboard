GitHub Pages setup
Step-by-step guide to publish this dashboard as a live website
What you'll get

A permanent URL like yourname.github.io/spa-dashboard that you can bookmark, share with your team, and update whenever you upload a new CSV. Free, no monthly fees, no server to manage.

Step-by-step instructions
1
Create a free GitHub account
Go to github.com → click "Sign up" → enter your email, password, and username. Verify your email. Free plan is all you need.
2
Create a new repository
After logging in, click the + icon (top right) → "New repository". Name it spa-dashboard. Set it to Public. Check "Add a README file". Click "Create repository".
3
Upload your dashboard files
Inside your new repository, click Add file → Upload files. Drag and drop the files you downloaded from Claude (index.html). Click "Commit changes".
4
Enable GitHub Pages
In your repository, go to Settings → Pages (left sidebar). Under "Source", select Deploy from a branch. Branch: main, folder: / (root). Click Save. Wait ~2 minutes.
5
Your dashboard is live
GitHub will show your URL at the top of the Pages settings page: https://[your-username].github.io/spa-dashboard. Bookmark it. Share it with your team.
6
Updating data going forward
Each month: export the SPA report CSV from Salesforce → open your dashboard URL → go to "Upload data" tab → drop in the new CSV. The dashboard refreshes instantly in your browser. To permanently update the hosted version, re-upload index.html to GitHub (we'll automate this later).
Need help?

If you get stuck on any step, take a screenshot and share it with Claude — we can troubleshoot together and update the files as needed.
